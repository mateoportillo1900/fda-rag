📦 src/fda_rag/ingestion/
Parses XML → chunks text → embeds → stores in Neon
parser.py → extracts 8 section types from XML
chunker.py → splits into 1500-char windows
loader.py → calls Voyage AI + writes to Neon
__init__.py → exports ParsedLabel, Chunk
↓ calls Voyage AI ↓
🌐 Voyage AI — embed
Converts each text chunk into a float[1024] vector
model: voyage-3
input_type: "document"
output: 1024-dim vector per chunk
↓ INSERT INTO drug_chunks ↓
🗄️ Neon — Postgres + pgvector
Cloud database. Stores text + embeddings. Connected via DATABASE_URL in .env
table: drug_chunks
columns: drug_name, section_name, chunk_text
column: embedding vector(1024)
index: HNSW (fast nearest-neighbour search)
How the offline path runs
run_ingestion.py
  calls parse_label()
  calls chunk_label()
  calls embed_chunks() → Voyage AI
  calls store_chunks() → Neon
Shared database
Offline path writes to Neon
Online path reads from Neon

Both use DATABASE_URL from .env
How the online path runs
ui/ → HTTP POST /query
  → api/main.py
  → agent.invoke()
    → retrieve() → Neon
    → rerank() → Voyage AI
    → generate() → Groq / Llama 3.3
  ← { answer, sources }

🌐 Voyage AI — query + rerank
Embeds the user question, then reranks the top-20 results
embed: voyage-3, input_type: "query"
rerank: rerank-2, top_n: 5
↓ top 5 chunks → generate_node ↓
🌐 Groq — Llama 3.3 70B
Reads the chunks + question, writes a cited answer
model: llama-3.3-70b-versatile
system prompt: cite sources, no hallucination
output: plain-text answer

Quick-reference — key commands

Command What it does When to run
python scripts/download_dailymed.py --sample Downloads 10 FDA drug label XML files into data/sample/xml/ Once, before ingestion
python scripts/migrate.py Creates the drug_chunks table + HNSW index in Neon Once, first-time setup
python scripts/run_ingestion.py Parses XML → chunks → embeds → stores all drug labels in Neon Once per data load
uvicorn fda_rag.api.main:app --reload --port 8000 Starts the FastAPI server at localhost:8000 During development / serving
pytest tests/ -v Runs all tests (smoke + ingestion + retrieval) After any code change
ruff check src/ tests/ Lints the codebase for errors and style issues Before committing
