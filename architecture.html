The two separate lifetimes

① Offline — Ingestion

Runs once (or when new FDA data arrives). Parses XML files, chunks the text, generates embeddings, and stores everything in Postgres. This must finish before anyone can ask a question.

Triggered manually via scripts/

② Online — Query

Runs on every user question. Embeds the question, searches Postgres for relevant chunks, reranks them, then calls Claude to write a grounded, cited answer.

Triggered by user typing in the UI

What each layer imports / calls

ui/
Does NOT import other subpackages.
Calls api/ over HTTP only.
This means you can run the UI and API on separate machines.
api/
Imports agent/
Exposes HTTP endpoints that the UI calls.
agent/
Imports retrieval/
Calls Google Gemini API (external)
Logged by LangSmith (passive)
retrieval/
Calls Voyage AI API (embed + rerank)
Queries Postgres via psycopg
Does NOT import agent/ or api/
ingestion/
Calls Voyage AI API (embed chunks)
Writes to Postgres via psycopg
Completely separate from the online path
eval/
Imports agent/ (runs it)
Uses RAGAS (external library)
Never called in production

Where each environment variable is used

Variable Used by When it's needed
DATABASE_URL ingestion/, retrieval/, tests/ Offline (writing chunks) + Online (querying chunks) + smoke test
VOYAGE_API_KEY ingestion/ + retrieval/ Offline (embed chunks) + Online (embed question + rerank results)
GEMINI_API_KEY agent/ — generate node Online only — called once per user question to generate the answer
LANGSMITH_API_KEY LangSmith SDK (auto-detected) Online — every agent call is traced if LANGCHAIN_TRACING_V2=true
LANGSMITH_PROJECT LangSmith SDK Groups all traces under one named project in the dashboard
LANGCHAIN_TRACING_V2 LangSmith SDK Set to true to turn tracing on; false to run silently
